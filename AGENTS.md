# Project Rules & Constraints

## 🛡️ Security & Privacy
- **DO NOT** read, open, or `cat` the `.env` file under any circumstances.
- **DO NOT** output environment variables or database connection strings in any task logs, implementation plans, or artifacts.
- If an environment variable is missing, **STOP** and ask the user to provide it or verify it manually.
- Always verify that `.env` is listed in `.gitignore` before performing any Git-related tasks.
